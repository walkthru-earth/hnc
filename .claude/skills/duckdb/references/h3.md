# H3 Hexagonal Grid for DuckDB

Indexes points and geometries to hexagons using [Uber's H3](https://h3geo.org) discrete global grid system.

Source: [isaacbrodsky/h3-duckdb](https://github.com/isaacbrodsky/h3-duckdb)

## Step 0 -- Discover available h3_* functions (MUST run first)

Before writing any H3 query, run this to get the latest function signatures directly from the engine:

```sql
INSTALL h3 FROM community;
LOAD h3;
SELECT function_name, function_type, return_type, parameters, parameter_types
FROM duckdb_functions()
WHERE function_name LIKE 'h3_%'
ORDER BY function_name, return_type;
```

This is the **authoritative source**. It reflects the exact version installed, including parameter names, types, and overloads. H3 does not provide built-in `description` values, so use the category guide below for semantics, then verify exact signatures with the query above. To look up a specific function: `AND function_name = 'h3_grid_disk'`.

## Gotchas

- **H3 uses LAT, LNG order** (not lon, lat like most GIS tools). `h3_latlng_to_cell(latitude, longitude, resolution)`.
- **Parameter names are generic** (`col0, col1, ...`) in `duckdb_functions()`. Rely on the category guide for semantics.
- **h3_great_circle_distance** takes 5 args: `(lat1, lng1, lat2, lng2, unit)` where unit is `'km'`, `'m'`, or `'rads'`.
- **Most functions have 3 overloads**: UBIGINT, BIGINT, and VARCHAR input. UBIGINT is preferred (best performance).
- **Polygon fill functions**: `_experimental` variants use a newer, potentially faster algorithm.

## Function categories (74 functions)

| Category | Key functions | Notes |
|----------|--------------|-------|
| **Coordinate conversion** | `h3_latlng_to_cell`, `h3_cell_to_latlng`, `h3_cell_to_lat/lng` | lat/lng in, cell out (or reverse) |
| **Cell geometry** | `h3_cell_to_boundary_wkt`, `h3_cell_to_boundary_wkb` | WKT/WKB output for visualization |
| **Hierarchy** | `h3_cell_to_parent`, `h3_cell_to_children`, `h3_cell_to_center_child`, `h3_compact_cells`, `h3_uncompact_cells` | Parent/child traversal, compaction |
| **Grid traversal** | `h3_grid_disk`, `h3_grid_ring`, `h3_grid_distance`, `h3_grid_path_cells` | Neighbors, rings, paths. `_unsafe` variants skip pentagon checks |
| **Directed edges** | `h3_cells_to_directed_edge`, `h3_directed_edge_to_cells`, `h3_are_neighbor_cells` | Edge adjacency, boundary WKT/WKB |
| **Vertices** | `h3_cell_to_vertex`, `h3_cell_to_vertexes`, `h3_vertex_to_latlng` | Corner point access |
| **Measurement** | `h3_cell_area`, `h3_get_hexagon_area_avg`, `h3_edge_length`, `h3_great_circle_distance` | Area, edge length, haversine |
| **Polygon conversion** | `h3_polygon_wkt_to_cells`, `h3_polygon_wkb_to_cells`, `h3_cells_to_multi_polygon_wkt/wkb` | Polygon fill and reconstruction. `_experimental` variants use a newer algorithm |
| **Metadata/validation** | `h3_get_resolution`, `h3_is_valid_cell`, `h3_is_pentagon`, `h3_get_num_cells`, `h3_get_res0_cells` | Validation and global stats |
| **String conversion** | `h3_string_to_h3`, `h3_h3_to_string`, `h3_latlng_to_cell_string` | VARCHAR variants (append `_string` to most functions) |

## Common patterns

```sql
-- Index points to H3 cells
SELECT h3_latlng_to_cell(latitude, longitude, 9) AS cell_id, COUNT(*) AS cnt
FROM my_points GROUP BY cell_id ORDER BY cnt DESC;

-- Integer API (preferred performance)
SELECT h3_cell_to_latlng(586265647244115967);

-- Neighbors within 2 grid steps
SELECT unnest(h3_grid_disk(h3_latlng_to_cell(40.7128, -74.006, 9), 2)) AS neighbor;

-- Fill a polygon with H3 cells
SELECT unnest(h3_polygon_wkt_to_cells('POLYGON((-74 40, -73 40, -73 41, -74 41, -74 40))', 7)) AS cell;
```
