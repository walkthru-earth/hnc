# A5 Pentagonal Grid for DuckDB

Millimeter-accurate, equal-area geospatial index based on a geodesic pentagonal grid. OGC DGGS compliant.

Source: [Query-farm/a5](https://github.com/Query-farm/a5)

## Step 0 -- Discover available a5_* functions (MUST run first)

Before writing any A5 query, run this to get the latest function signatures directly from the engine:

```sql
INSTALL a5 FROM community;
LOAD a5;
SELECT function_name, function_type, return_type,
       parameters, parameter_types, description
FROM duckdb_functions()
WHERE function_name LIKE 'a5_%'
ORDER BY function_name;
```

This is the **authoritative source**. A5 provides built-in descriptions, named parameters, and typed signatures for every function. Use them to understand exact usage before composing queries. To look up a specific function: `AND function_name = 'a5_grid_disk'`.

## Gotchas

- **A5 uses LON, LAT order**: `a5_lonlat_to_cell(longitude, latitude, resolution)`. Standard GIS order.
- **Pentagonal cells** (not hexagonal). Each cell has 4-5 neighbors, not 6.
- **`a5_cell_to_children` has 2 overloads**: with target_res (skip levels) or without (immediate children only).
- **`a5_cell_to_boundary` has 3 overloads**: `(cell)`, `(cell, closed_ring)`, `(cell, closed_ring, segments)`.
- **`a5_spherical_cap` radius is in meters**, not degrees.
- **Cell IDs are UBIGINT**. No BIGINT or VARCHAR overloads (unlike H3).

## Function categories (18 functions)

| Category | Key functions | Notes |
|----------|--------------|-------|
| **Coordinate conversion** | `a5_lonlat_to_cell(lon, lat, res)` | Returns UBIGINT cell ID |
| **Cell geometry** | `a5_cell_to_lonlat`, `a5_cell_to_boundary`, `a5_cell_to_spherical` | Center, boundary vertices, spherical coords |
| **Hierarchy** | `a5_cell_to_parent`, `a5_cell_to_children` | Coarser/finer resolution traversal |
| **Traversal** | `a5_grid_disk`, `a5_grid_disk_vertex`, `a5_spherical_cap` | Edge/vertex adjacency, radius search (meters) |
| **Compaction** | `a5_compact`, `a5_uncompact` | Merge siblings into parents, expand to target res |
| **Metadata** | `a5_cell_area`, `a5_get_resolution`, `a5_get_num_cells`, `a5_get_res0_cells`, `a5_get_num_children` | Area in m2, cell counts |
| **String conversion** | `a5_hex_to_u64`, `a5_u64_to_hex` | Hex string to/from UBIGINT |

## Common patterns

```sql
-- Spatial aggregation
SELECT a5_lonlat_to_cell(longitude, latitude, 15) AS cell_id, COUNT(*) AS cnt
FROM my_table GROUP BY cell_id ORDER BY cnt DESC;

-- Generate GeoJSON (requires spatial extension)
LOAD spatial;
SELECT ST_AsGeoJSON(ST_MakePolygon(ST_MakeLine(
    list_transform(a5_cell_to_boundary(a5_lonlat_to_cell(-3.7037, 40.41677, 10)),
    x -> ST_Point(x[1], x[2]))))) AS geojson;

-- Neighbors within 2 edge-steps
SELECT unnest(a5_grid_disk(a5_lonlat_to_cell(-74.006, 40.7128, 15), 2)) AS neighbor;

-- All cells within 5km radius
SELECT unnest(a5_spherical_cap(a5_lonlat_to_cell(-74.006, 40.7128, 15), 5000.0)) AS cell;
```
